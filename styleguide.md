**[&nbsp;TEXT&nbsp;]**  
**`<span style="text-transform: uppercase">UPPERCASE</span>` ALL TEXT IN UPPERCASE TO MATCH FORMATTING IN *MYSTERY HOUSE*.  
`**BOLD**` ALL TEXT IN BOLD.  
`*ITALICS*` TEXT IN ITALICS FOR OPTIONAL ACTIONS.  
`<ins>UNDERLINE</ins>` UNDERLINED TEXT FOR COMMANDS TO INPUT.  
`` / `` OPEN SLASHES / VIRGULES.  
`` -- `` OPEN EM DASHES STYLED AS 2 HYPHENS.  
`` - `` OPEN EN DASHES STYLED AS 1 HYPHEN.  
`-` CLOSED HYPHENS.  
`()` CLOSED PARENTHESES.  
`''` STRAIGHT SINGLE QUOTES.  
`""` STRAIGHT DOUBLE QUOTES.  
`0 - 100` 0 - 100 WRITTEN NUMERICALLY.  
`>` ANGLE BRACKETS AT THE BEGINNING OF EACH PARAGRAPH OR LINE FOR BLOCK QUOTES.**

**[&nbsp;BODY&nbsp;]**  
**`<!--GLOBAL NOTE-->` ENDS OF PARAGRAPHS AND LINES ARE FORMATTED WITH 2 SPACES BEFORE CARRIAGE RETURN PER MARKDOWN STYLE.  
`<p>PARAGRAPH STYLE</p>` PARAGRAPH STYLE IS FLUSH LEFT WITH LINE BREAKS BETWEEN EACH PARAGRAPH AND AT THE END OF EACH SUBSECTION.  
`--------------- ENTER COMMAND? <ins>1-WORD DIRECTION OR 2-WORD ACTION</ins>` COMMAND PROMPT AND INPUT ON SAME LINE TO FOLLOW LAYOUT IN GAME.  
`<ul>LIST STYLE</ul>` LIST STYLE IS FLUSH LEFT WITH SEPARATE LINES FOR EACH ENTRY AND A LINE BREAK AT THE END.**

**[&nbsp;FORMATTING&nbsp;]**  
**`<!--GLOBAL NOTE-->` HEADINGS, SUBHEADINGS, NAVIGATION HEADERS AND FOOTERS, HORIZONTAL RULES, AND SUBHEADERS ARE FORMATTED WITH 2 SPACES BEFORE CARRIAGE RETURN.  
`[&nbsp;HEADINGS&nbsp;]` HEADINGS IN BRACKETS FLANKED WITH HTML NONBREAKING SPACES.  
`[&nbsp;SUBHEADINGS&nbsp;]` SUBHEADINGS FOR SUBSECTIONS IN BRACKETS WITH HTML NONBREAKING SPACES.  
`[[&nbsp;NAVIGATION HEADERS&nbsp;]](/navigationheaders.md)` `[[&nbsp;NAVIGATION HEADERS&nbsp;]](#navigationheaders)` CONTENTS NAVIGATION HEADERS LINK TO CORRESPONDING PAGES AND SECTIONS.  
`[[&nbsp;NAVIGATION FOOTERS&nbsp;]](/navigationfooters.md)` CONTENTS NAVIGATION FOOTERS LINK TO PREVIOUS AND NEXT PAGES.  
`[[&nbsp;SECTION LINKS&nbsp;]](/sectionlinks.md#sectionlinks)` `<a name="sectionlinks">[&nbsp;SECTION LINKS&nbsp;]</a>` SECTION LINKS TAGGED WITH CUSTOM ANCHORS.  
`<a name="anchornames"></a>` ANCHOR NAMES ARE LOWERCASE AND ALL ONE WORD WITHOUT HYPHENS.  
`.md` MARKDOWN FILE NAMES ARE LOWERCASE AND ALL ONE WORD WITHOUT HYPHENS.  
`.git` GIT FILE NAMES ARE LOWERCASE AND ALL ONE WORD WITHOUT HYPHENS.  
`---` RESPONSIVE HORIZONTAL RULES ABOVE AND BELOW HEADINGS, NAVIGATION HEADERS AND FOOTERS, AND BODY TEXT (10 PER SECTION).  
`####` MARKDOWN SUBHEADERS INCLUDED FOR SPACING BEFORE AND AFTER HEADINGS, NAVIGATION HEADERS AND FOOTERS, BODY TEXT, AND SUBSECTIONS.**
