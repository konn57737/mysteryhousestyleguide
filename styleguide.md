**[&nbsp;TEXT&nbsp;]**  
**`<span style="text-transform: uppercase">UPPERCASE</span>` ALL TEXT IS UPPERCASE TO MATCH TYPE IN *MYSTERY HOUSE*.  
`**BOLD**` ALL TEXT IS BOLD.  
`*ITALICS*` OPTIONAL ACTIONS ARE ITALICIZED.  
`<ins>UNDERLINING</ins>` COMMANDS TO INPUT ARE UNDERLINED.  
`` / `` SLASHES / VIRGULES ARE OPEN.  
`` -- `` EM DASHES ARE OPEN AND STYLED AS 2 HYPHENS.  
`` - `` EN DASHES ARE OPEN AND STYLED AS 1 HYPHEN.  
`-` HYPHENS ARE CLOSED.  
`()` PARENTHESES ARE CLOSED.  
`''` SINGLE QUOTES AND APOSTROPHES ARE FORMATTED WITH STRAIGHT QUOTATION MARKS.  
`""` DOUBLE QUOTES ARE FORMATTED WITH STRAIGHT QUOTATION MARKS.  
`0 - 100` 0 - 100 ARE WRITTEN NUMERICALLY.  
`>` ANGLE BRACKETS ARE INCLUDED AT THE BEGINNING OF EACH PARAGRAPH OR LINE TO SET OFF AND INDENT TEXT FOR BLOCK QUOTES.**

**[&nbsp;HEADINGS&nbsp;]**  
**`<!--GLOBAL NOTE-->` HEADINGS, SUBHEADINGS, AND NAVIGATION HEADERS AND FOOTERS ARE FORMATTED WITH 2 SPACES BEFORE CARRIAGE RETURN.  
`[&nbsp;HEADINGS&nbsp;]` HEADINGS FOR TITLES AND SECTIONS ARE IN BRACKETS FLANKED WITH HTML NONBREAKING SPACES.  
`[&nbsp;SUBHEADINGS&nbsp;]` SUBHEADINGS FOR SUBSECTIONS OR LOCATIONS IN GAMEPLAY ARE IN BRACKETS WITH HTML NONBREAKING SPACES.  
`[[&nbsp;NAVIGATION HEADERS&nbsp;]](/navigationheaders.md)` `[[&nbsp;NAVIGATION HEADERS&nbsp;]](#navigationheaders)` TABLE OF CONTENTS NAVIGATION HEADERS LINK TO CORRESPONDING PAGES AND SECTIONS.  
`[[&nbsp;NAVIGATION FOOTERS&nbsp;]](/navigationfooters.md)` TABLE OF CONTENTS NAVIGATION FOOTERS LINK TO PREVIOUS AND NEXT PAGES.  
`[[&nbsp;SECTION LINKS&nbsp;]](/sectionlinks.md#sectionlinks)` `<a name="sectionlinks">[&nbsp;SECTION LINKS&nbsp;]</a>` SECTION LINKS ARE TAGGED WITH CUSTOM ANCHORS.**

**[&nbsp;BODY&nbsp;]**  
**`<!--GLOBAL NOTE-->` ENDS OF PARAGRAPHS AND LINES ARE FORMATTED WITH 2 SPACES BEFORE CARRIAGE RETURN PER MARKDOWN STYLE.  
`<p>PARAGRAPH STYLE</p>` PARAGRAPH STYLE IS FLUSH LEFT WITH LINE BREAKS BETWEEN EACH PARAGRAPH AND AT THE END OF EACH SUBSECTION.  
`<ul>LIST STYLE</ul>` LIST STYLE IS FLUSH LEFT WITH SEPARATE LINES FOR EACH ENTRY AND A LINE BREAK AT THE END.  
`--------------- ENTER COMMAND? <ins>1-WORD DIRECTION OR 2-WORD ACTION</ins>` COMMAND PROMPT AND INPUT ARE WRITTEN ON THE SAME LINE TO FOLLOW LAYOUT IN GAME. "ENTER COMMAND?" IS PRECEDED BY 15 HYPHENS AND 1-WORD DIRECTION OR 2-WORD ACTION TO INPUT IS UNDERLINED.**

**[&nbsp;FORMATTING&nbsp;]**  
**`<!--GLOBAL NOTE-->` HORIZONTAL RULES AND SUBHEADERS ARE FORMATTED WITH 2 SPACES BEFORE CARRIAGE RETURN.  
`---` RESPONSIVE HORIZONTAL RULES INCLUDED ABOVE AND BELOW HEADINGS, NAVIGATION HEADERS AND FOOTERS, AND BODY TEXT (10 PER SECTION).  
`####` MARKDOWN SUBHEADERS FOR SPACING BEFORE AND AFTER HEADINGS, NAVIGATION HEADERS AND FOOTERS, BODY TEXT, AND SUBSECTIONS.  
`<a name="anchornames"></a>` ANCHOR NAMES ARE LOWERCASE AND ALL ONE WORD WITHOUT HYPHENS.  
`.md` MARKDOWN FILE NAMES ARE LOWERCASE AND ALL ONE WORD WITHOUT HYPHENS.  
`.git` GIT FILE NAMES ARE LOWERCASE AND ALL ONE WORD WITHOUT HYPHENS.**

**[&nbsp;USAGES&nbsp;]**  
**INDEFINITE ARTICLES FOR CLARITY `(CMOS 5.74)`.   
OMISSIVE, POSSESSIVE, AND PLURAL APOSTROPHES `(CMOS 6.124)`.  
CONSISTENT ORDERING IN RELATED LISTS `(CMOS 6.138)`.  
SERIAL / SERIES COMMAS `(CMOS 6.19)`.  
COMMAS AFTER INTRODUCTORY PHRASES `(CMOS 6.36)`.  
COMMAS AFTER CONJUNCTIVE ADVERBS `(CMOS 6.52)`.  
PLURAL FOR COMPOUND NOUNS `(CMOS 7.7)`.  
SPELLING FOR COMMONLY USED WORDS MATCHES MERRIAM-WEBSTER `(M-W 2025)`.**
