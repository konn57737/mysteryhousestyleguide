**[&nbsp;TEXT&nbsp;]**  
**`<!--UPPERCASE-->` ALL TEXT IS UPPERCASE TO MATCH TYPE IN *MYSTERY HOUSE*.  
`**BOLD**` ALL TEXT IS BOLD.  
`*ITALIC*` ALL TEXT FOR OPTIONAL ACTIONS, ALTERNATE ENDINGS, AND POSSIBLE INVENTORY ITEMS IS ITALICIZED.  
`<ins>UNDERLINE</ins>` COMMANDS TO INPUT ARE UNDERLINED.  
`ZERO - ONE HUNDRED` ZERO - ONE HUNDRED WHOLE NUMBERS ARE SPELLED OUT.  
`0 - 100` 0 - 100 NUMERALS ARE USED IN MATHEMATICAL EXPRESSIONS AND DECIMALS, YEARS, ADDRESSES, AND WITH SYMBOLS.  
` ... ` ELLIPSES ARE OPEN WITH SPACES BEFORE AND AFTER THREE CLOSED PERIODS.    
`` / `` SLASHES / VIRGULES ARE OPEN.  
`` -- `` EM DASHES ARE OPEN AND STYLED AS TWO HYPHENS.  
`` - `` EN DASHES ARE OPEN AND STYLED AS ONE HYPHEN.  
`-` HYPHENS ARE CLOSED.  
`()` PARENTHESES ARE CLOSED.  
` # ` SYMBOLS ARE OPEN.  
`''` SINGLE QUOTES AND APOSTROPHES ARE FORMATTED WITH STRAIGHT QUOTATION MARKS.  
`""` DOUBLE QUOTES ARE FORMATTED WITH STRAIGHT QUOTATION MARKS.  
`>` ANGLE BRACKETS ARE INCLUDED AT THE BEGINNING OF EACH PARAGRAPH OR LINE TO SET OFF AND INDENT TEXT FOR BLOCK QUOTES AND NOTES / SOURCES ARE ENCLOSED IN BRACKETS WITH HTML NONBREAKING SPACES.**

**[&nbsp;HEADINGS&nbsp;]**  
**`[&nbsp;HEADING&nbsp;]` HEADINGS FOR TITLES AND SECTIONS ARE IN BRACKETS FLANKED WITH HTML NONBREAKING SPACES AND ARE TAGGED WITH MANUAL ANCHORS.  
`[&nbsp;SUBHEADING&nbsp;]` SUBHEADINGS FOR SUBSECTIONS ARE IN BRACKETS WITH HTML NONBREAKING SPACES.  
`[[&nbsp;NAVIGATION HEADER&nbsp;]](/filename.md)` `[[&nbsp;NAVIGATION HEADER&nbsp;]](#sectionlink)` NAVIGATION HEADERS FOR TABLE OF CONTENTS LINK TO CORRESPONDING PAGES AND SECTIONS.  
`[[&nbsp;NAVIGATION FOOTER&nbsp;]](/filename.md)` NAVIGATION FOOTERS FOR CONTENTS LINK TO PREVIOUS AND NEXT PAGES.  
`[[&nbsp;SECTION LINK&nbsp;]](/filename.md#sectionlink)` `<a name="anchorname">[&nbsp;SECTION LINK&nbsp;]</a>` SECTION LINKS FOR NAVIGATION HEADERS AND SUSBSECTIONS ARE TAGGED WITH MANUAL ANCHORS.**

**[&nbsp;BODY&nbsp;]**  
**`[&nbsp;READ ME&nbsp;]` `[&nbsp;INTRODUCTION&nbsp;]` `[&nbsp;INSTRUCTIONS&nbsp;]` PARAGRAPH STYLE IS FLUSH LEFT WITH LINE BREAKS BETWEEN EACH PARAGRAPH.  
`[&nbsp;WALKTHROUGH&nbsp;]` `[&nbsp;ALTERNATE ENDINGS&nbsp;]` SUBSECTION PARAGRAPH STYLE IS FLUSH LEFT WITH CARRIAGE RETURNS AFTER SUBHEADINGS AND ALL LINES (DESCRIPTIONS, COMMAND PROMPTS AND INPUTS, RESPONSES, AND OPTIONAL ACTIONS) AND LINE BREAKS BEFORE AND AFTER BLOCK QUOTES. CONTINUOUS SUBSECTION PARAGRAPHS IN `[&nbsp;WALKTHROUGH&nbsp;]` ARE SEPARATED WITH SUBHEADERS AND NONCONTINUOUS SUBSECTION PARAGRAPHS IN `[&nbsp;ALTERNATE ENDINGS&nbsp;]` ARE SEPARATED WITH HORIZONTAL RULES.   
`[&nbsp;INTRODUCTION&nbsp;]` `[&nbsp;INSTRUCTIONS&nbsp;]` `[&nbsp;INVENTORY&nbsp;]` LIST STYLE IS FLUSH LEFT WITH CARRIAGE RETURNS AFTER EACH LINE.**

**[&nbsp;FORMATTING&nbsp;]**  
**`Â¶` HEADINGS, SUBHEADINGS, NAVIGATION HEADERS AND FOOTERS, PARAGRAPHS, LINES, HORIZONTAL RULES, AND SUBHEADERS ARE FORMATTED WITH TWO SPACES BEFORE CARRIAGE RETURN PER MARKDOWN STYLE.  
`---` RESPONSIVE HORIZONTAL RULES ARE INCLUDED ABOVE AND BELOW HEADINGS, NAVIGATION HEADERS AND FOOTERS, BODY TEXT, AND NONCONTINUOUS SUBSECTIONS (TEN PER SECTION).  
`####` MARKDOWN SUBHEADERS WITH SURROUNDING LINE BREAKS ADDED FOR SPACING BEFORE AND AFTER HEADINGS, NAVIGATION HEADERS AND FOOTERS, BODY TEXT, AND CONTINUOUS SUBSECTIONS.  
`--------------- COMMAND PROMPT? <ins>COMMAND INPUT</ins>` COMMAND PROMPTS AND INPUTS ARE WRITTEN ON THE SAME LINE AND ARE PRECEDED BY FIFTEEN HYPHENS TO FOLLOW LAYOUT IN GAME.  
`<a name="anchorname"></a>` ANCHOR NAMES ARE LOWERCASE AND ALL ONE WORD WITHOUT HYPHENS.  
`filename.md` MARKDOWN FILE NAMES ARE LOWERCASE AND ALL ONE WORD WITHOUT HYPHENS.  
`filename.git` GIT FILE NAMES ARE LOWERCASE AND ALL ONE WORD WITHOUT HYPHENS.**
