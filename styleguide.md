**[&nbsp;TEXT&nbsp;]**  
**`<!--UPPERCASE-->` ALL TEXT IS UPPERCASE TO MATCH TYPE IN *MYSTERY HOUSE*.  
`**BOLD**` ALL TEXT IS BOLD.  
`*ITALIC*` ALL TEXT FOR OPTIONAL ACTIONS, ADDITIONAL NOTES, AND TITLES ARE ITALICIZED.  
`<ins>UNDERLINE</ins>` COMMANDS TO INPUT ARE UNDERLINED.  
`` / `` SLASHES / VIRGULES ARE OPEN.  
`` -- `` EM DASHES ARE OPEN AND STYLED AS 2 HYPHENS.  
`` - `` EN DASHES ARE OPEN AND STYLED AS 1 HYPHEN.  
`-` HYPHENS ARE CLOSED.  
`()` PARENTHESES ARE CLOSED.  
`''` SINGLE QUOTES AND APOSTROPHES ARE FORMATTED WITH STRAIGHT QUOTATION MARKS.  
`""` DOUBLE QUOTES ARE FORMATTED WITH STRAIGHT QUOTATION MARKS.  
`0 - 100` 0 - 100 ARE WRITTEN NUMERICALLY.  
`>` ANGLE BRACKETS ARE INCLUDED AT THE BEGINNING OF EACH PARAGRAPH OR LINE TO SET OFF AND INDENT TEXT FOR BLOCK QUOTES.**

**[&nbsp;HEADINGS&nbsp;]**  
***HEADINGS, SUBHEADINGS, AND NAVIGATION HEADERS AND FOOTERS ARE FORMATTED WITH 2 SPACES BEFORE CARRIAGE RETURN PER MARKDOWN STYLE.*  
`[&nbsp;HEADING&nbsp;]` HEADINGS FOR TITLES AND SECTIONS ARE IN BRACKETS FLANKED WITH HTML NONBREAKING SPACES.  
`[&nbsp;SUBHEADING&nbsp;]` SUBHEADINGS FOR SUBSECTIONS ARE IN BRACKETS WITH HTML NONBREAKING SPACES.  
`[[&nbsp;NAVIGATION HEADER&nbsp;]](/filename.md)` `[[&nbsp;NAVIGATION HEADER&nbsp;]](#sectionlink)` NAVIGATION HEADERS FOR TABLE OF CONTENTS LINK TO CORRESPONDING PAGES AND SECTIONS.  
`[[&nbsp;NAVIGATION FOOTER&nbsp;]](/filename.md)` NAVIGATION FOOTERS FOR VIEWING CONTENTS IN ORDER LINK TO PREVIOUS AND NEXT PAGES.  
`[[&nbsp;SECTION LINK&nbsp;]](/filename.md#sectionlink)` `<a name="anchorname">[&nbsp;SECTION LINK&nbsp;]</a>` SECTION LINKS ARE TAGGED WITH CUSTOM ANCHORS.**

**[&nbsp;BODY&nbsp;]**  
***ENDS OF PARAGRAPHS AND LINES ARE FORMATTED WITH 2 SPACES BEFORE CARRIAGE RETURN PER MARKDOWN STYLE.*  
`[&nbsp;READ ME&nbsp;]` `[&nbsp;INTRODUCTION&nbsp;]` `[&nbsp;INSTRUCTIONS&nbsp;]` PARAGRAPH STYLE IS FLUSH LEFT WITH LINE BREAKS BETWEEN EACH PARAGRAPH.  
`[&nbsp;WALKTHROUGH&nbsp;]` `[&nbsp;ALTERNATE ENDINGS&nbsp;]` SUBSECTION PARAGRAPH STYLE IS FLUSH LEFT WITH CARRIAGE RETURNS AFTER SUBHEADINGS AND ALL LINES (DESCRIPTIONS OF SCENES AND OBJECTS, COMMAND PROMPTS AND INPUTS, RESPONSES TO INPUTTED COMMANDS , AND OPTIONAL ACTIONS) AND WITH LINE BREAKS BEFORE AND AFTER BLOCK QUOTES. CONTINUOUS SUBSECTION PARAGRAPHS IN `[&nbsp;WALKTHROUGH&nbsp;]` ARE SEPARATED WITH SUBHEADERS AND NONCONTINUOUS IN `[&nbsp;ALTERNATE ENDINGS&nbsp;]` ARE SEPARATED WITH HORIZONTAL RULES.   
`[&nbsp;INTRODUCTION&nbsp;]` `[&nbsp;INSTRUCTIONS&nbsp;]` `[&nbsp;INVENTORY&nbsp;]` LIST STYLE IS FLUSH LEFT WITH CARRIAGE RETURNS FOR EACH LINE.**

**[&nbsp;FORMATTING&nbsp;]**  
***HORIZONTAL RULES AND SUBHEADERS ARE FORMATTED WITH 2 SPACES BEFORE CARRIAGE RETURN PER MARKDOWN STYLE.*  
`---` RESPONSIVE HORIZONTAL RULES INCLUDED ABOVE AND BELOW HEADINGS, NAVIGATION HEADERS AND FOOTERS, BODY TEXT, AND NONCONTINUOUS SUBSECTIONS (10 PER SECTION).  
`####` MARKDOWN SUBHEADERS WITH SURROUNDING LINE BREAKS ADDED FOR SPACING BEFORE AND AFTER HEADINGS, NAVIGATION HEADERS AND FOOTERS, BODY TEXT, AND CONTINUOUS SUBSECTIONS.  
`--------------- COMMAND PROMPT? <ins>COMMAND INPUT</ins>` COMMAND PROMPTS AND INPUTS ARE WRITTEN ON THE SAME LINE TO FOLLOW LAYOUT IN GAME. COMMAND PROMPTS ARE PRECEDED BY 15 HYPHENS AND COMMAND INPUTS ARE 1-WORD DIRECTIONS OR 2-WORD ACTIONS.  
`<a name="anchorname"></a>` ANCHOR NAMES ARE LOWERCASE AND ALL ONE WORD WITHOUT HYPHENS.  
`filename.md` MARKDOWN FILE NAMES ARE LOWERCASE AND ALL ONE WORD WITHOUT HYPHENS.  
`filename.git` GIT FILE NAMES ARE LOWERCASE AND ALL ONE WORD WITHOUT HYPHENS.**
